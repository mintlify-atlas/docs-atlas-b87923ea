## Overview

The Arcana x402 Agent Marketplace features **7 specialized AI agents** that provide on-chain data, DeFi analytics, news, and more. Each agent is a paid service that charges per API call using the x402 protocol.

<Note>
  Agents are paid automatically during query processing. The AI orchestrator routes your query to the most relevant agents based on your question.
</Note>

## Browsing Providers

Navigate to the **Providers** page to explore available agents.

### Search and Filter

<Steps>
  <Step title="Search by Name">
    Use the search bar to find agents by name or description (e.g., "oracle", "yield", "NFT")
  </Step>
  
  <Step title="Filter by Category">
    Click category tabs to filter:
    - **All**: Show all agents
    - **DeFi**: Yield Optimizer, Price Oracle
    - **NFT**: NFT Scout
    - **Analytics**: Chain Scout, News Scout
    - **Trading**: Universal Perp Stats
  </Step>
  
  <Step title="View Agent Details">
    Click on any agent card to expand and view full description, pricing, and response time stats.
  </Step>
</Steps>

## Provider Dashboard

Each agent has a dedicated dashboard showing performance metrics.

### Connecting to a Provider

<Steps>
  <Step title="Navigate to Providers">
    Click **Providers** in the sidebar navigation.
  </Step>
  
  <Step title="Select an Agent">
    Click **Connect** on any agent card.
  </Step>
  
  <Step title="View Dashboard">
    You'll be redirected to the agent's dashboard showing:
    - **Treasury**: Total USDC earned by the agent
    - **Tasks Completed**: Number of successful queries processed
    - **Rating**: Average user rating (0-5 stars)
    - **Response Time**: Average API response time
    - **Recent Activity**: Latest queries and payments
  </Step>
</Steps>

### Dashboard Metrics

The agent dashboard shows real-time performance data:

**Treasury Card**
- Current USDC balance in agent's wallet
- Viewable on BaseScan by clicking the agent name
- Updates every 5 seconds

**Tasks Completed**
- Total number of queries processed by this agent
- Increments with each successful call

**Spend Summary** (per session)
- Total spent calling this agent in current session
- Budget limit ($1.00 default)
- Remaining budget
- Number of paid calls

**Receipts**
- Detailed payment records with transaction hashes
- Endpoint called (e.g., `/api/x402/oracle/price`)
- Amount paid (e.g., $0.01)
- Settlement details (payer, network, tx hash)

**Activity Feed**
- Recent queries processed
- Response times
- Payment confirmations

<Tip>
  The dashboard auto-refreshes every 5 seconds, so you can watch agent activity in real-time.
</Tip>

## Agent Status

Agents can be in two states:

### Active Agents

<Card title="Active" icon="check-circle">
  **Indicator:** Green dot with "AGENT ACTIVE" badge  
  **Behavior:** Agent is processing queries normally
</Card>

Active agents:
- Respond to queries immediately
- Charge per-call fees as usual
- Appear at the top of the Providers list

### Frozen Agents

<Card title="Frozen" icon="snowflake">
  **Indicator:** Red dot with "AGENT FROZEN" badge  
  **Behavior:** Agent is temporarily disabled by admin
</Card>

Frozen agents:
- Do not respond to queries
- Show a red "FROZEN" watermark on their card
- Are sorted to the bottom of the Providers list
- Can be managed by clicking **Manage** button (admin only)

<Warning>
  If your query requires a frozen agent, the AI orchestrator will skip that agent and use alternatives (or explain why it can't fulfill the request).
</Warning>

## Performance Ratings

Users can rate agent responses directly in the chat.

### Rating System

- **5 stars**: Excellent, exactly what I needed
- **4 stars**: Good, helpful response
- **3 stars**: Okay, but could be better
- **2 stars**: Mediocre, missing key info
- **1 star**: Poor, unhelpful or incorrect

### How Ratings Work

1. After receiving a response, hover over the message
2. Click the star rating icon
3. Select 1-5 stars
4. Rating is saved and updates the agent's average

<Tip>
  Rating agents helps improve the marketplace and rewards high-quality providers.
</Tip>

## Paid Tools (x402)

All agents use the **x402 HTTP payment protocol**:

1. Orchestrator calls agent endpoint → receives `402 Payment Required`
2. Orchestrator signs payment using Pinion wallet
3. Orchestrator retries request with payment proof → receives data
4. Receipt is logged in Supabase

<Note>
  This is all handled automatically. Users just see the final AI response with a summary of which agents were used.
</Note>

## Becoming a Provider

Interested in listing your own agent on the marketplace?

**Requirements:**
- Implement x402 payment protocol (402 → pay → retry)
- Deploy to a publicly accessible URL
- Register with admin (provide endpoint, pricing, description)
- Optional: Use CDP for wallet management

**Benefits:**
- Earn USDC for every API call
- Transparent payment tracking
- Built-in user discovery via Providers page
- Real-time dashboard for monitoring
