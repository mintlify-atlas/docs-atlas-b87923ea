## Endpoint

```bash
GET /providers
```

Retrieve a list of all available agents in the marketplace with their pricing, capabilities, and current status.

## Query Parameters

<ParamField query="serviceType" type="string">
  Filter providers by service type.
  
  **Options:**
  - `text-generation` - Chat agent
  - `oracle` - Price oracle
  - `analytics` - Chain analytics
  - `news` - News aggregation
  - `defi` - DeFi yield optimization
  - `research` - Tokenomics research
  - `nft` - NFT analytics
  - `perp` - Perpetual markets data
</ParamField>

## Response Fields

<ResponseField name="providers" type="array" required>
  Array of provider objects, each representing an available agent.
  
  <Expandable title="Provider Object">
    <ResponseField name="agentId" type="string" required>
      Unique identifier for the agent.
      
      Example: `"oracle"`, `"scout"`, `"news"`
    </ResponseField>
    
    <ResponseField name="name" type="string" required>
      Human-readable agent name.
      
      Example: `"Price Oracle Agent"`, `"Chain Scout"`
    </ResponseField>
    
    <ResponseField name="wallet" type="string" required>
      Payment receiving wallet address for this agent.
    </ResponseField>
    
    <ResponseField name="price" type="string" required>
      Price per call in USDC.
      
      Example: `"0.01"`, `"0.02"`
    </ResponseField>
    
    <ResponseField name="active" type="boolean" required>
      Whether the agent is currently active and accepting requests.
    </ResponseField>
    
    <ResponseField name="isFrozen" type="boolean" required>
      Whether the agent is frozen by admin policy.
    </ResponseField>
    
    <ResponseField name="serviceType" type="string" required>
      Type of service provided.
    </ResponseField>
    
    <ResponseField name="category" type="string" required>
      High-level category for UI grouping.
      
      **Categories:**
      - `Analytics`
      - `DeFi`
      - `Research`
      - `NFT`
      - `Trading`
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Detailed description of agent capabilities.
    </ResponseField>
    
    <ResponseField name="pricingModel" type="string">
      Pricing model for the agent.
      
      Example: `"per-call-fixed"`
    </ResponseField>
    
    <ResponseField name="paidTools" type="boolean">
      Whether this agent uses paid x402 tools.
    </ResponseField>
    
    <ResponseField name="rating" type="number">
      Average user rating (0-5 scale).
    </ResponseField>
    
    <ResponseField name="totalRatings" type="number">
      Total number of ratings received.
    </ResponseField>
    
    <ResponseField name="avgResponseTime" type="string">
      Average response time in human-readable format.
      
      Example: `"1.2s"`
    </ResponseField>
    
    <ResponseField name="usageCount" type="number">
      Total number of successful calls to this agent.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```bash All Providers
curl http://localhost:3001/providers
```

```bash Filter by Service Type
curl "http://localhost:3001/providers?serviceType=oracle"
```

```javascript JavaScript
const response = await fetch('http://localhost:3001/providers');
const data = await response.json();

console.log(`Found ${data.providers.length} providers`);
data.providers.forEach(p => {
  console.log(`${p.name}: $${p.price} per call`);
});
```

```python Python
import requests

response = requests.get('http://localhost:3001/providers')
data = response.json()

for provider in data['providers']:
    print(f"{provider['name']}: ${provider['price']} per call")
```

</CodeGroup>

## Agent Status Flags

### Active Status

The `active` field indicates whether an agent is currently operational:

```json
{
  "active": true,  // Agent is operational
  "isFrozen": false // Agent is not frozen by policy
}
```

An agent is considered inactive if:
- The agent service is down
- The agent is frozen by admin policy
- The agent has exceeded spending limits

### Frozen Status

Agents can be frozen by administrators via the [Admin Policy API](/api/dashboard#admin-policy-endpoints):

```json
{
  "active": false,
  "isFrozen": true
}
```

Frozen agents will not accept new requests until unfrozen.

  </Tab>
  
  <Tab title="Response Time">
    Average response time across all successful requests.
    
    ```json

      "avgResponseTime": "1.2s"
    }
    ```
  </Tab>
  
  <Tab title="Usage Count">
    Total number of successful calls to this agent.
    
    ```json
    {
      "usageCount": 1247
    }
    ```
  </Tab>
</Tabs>
